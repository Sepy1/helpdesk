1. Tujuan & Cakupan

Aplikasi Helpdesk untuk pengelolaan tiket dukungan internal antara Cabang, IT, dan Vendor. Dokumen ini mendefinisikan kebutuhan fungsional berdasarkan kode dan migrasi saat ini.

2. Peran & Akses

  • IT: penuh atas manajemen tiket, ekspor, statistik, manajemen user.
  • CABANG: buat tiket, lihat tiket sendiri, komentar, unduh lampiran.
  • VENDOR: lihat tiket yang ditugaskan, isi tindak lanjut vendor.
  • ADMIN: tersedia di UI namun tidak ada pada enum DB saat ini.

Sumber kebenaran: kolom users.role (enum IT, CABANG, VENDOR; default CABANG).

3. Siklus Hidup Tiket

Status: OPEN → ON_PROGRESS → (ESKALASI_VENDOR) → (VENDOR_RESOLVED) → CLOSED

  • Cap waktu: taken_at, progress_at, vendor_followup_at, closed_at
  • Kolom terkait: eskalasi (TIDAK/VENDOR), progress_note, vendor_followup, closed_note, root_cause
  • Ketika CLOSED: form komentar & lampiran nonaktif; hapus komentar disembunyikan & diblok secara server.

4. Fitur Utama

4.1 Autentikasi & Navigasi

  • Home → login; Dashboard redirect sesuai peran.
  • Notifikasi: tandai satu/semua dibaca; dropdown responsif mobile.

4.2 Profil

  • Edit informasi (nama, email) & ubah password. Tampilan kartu; input putih.
  • Hapus akun dihilangkan dari UI.

4.3 CABANG

  • Buat tiket, lampiran opsional, komentar (kecuali CLOSED).

4.4 IT

  • Daftar tiket + filter (cari nomor/deskripsi/kategori; rentang tanggal; status); ekspor CSV/PNG.
  • Take/Release/Reopen/Close; progress; eskalasi vendor; assign vendor; vendor follow-up.
  • Statistik.

4.5 VENDOR

  • Lihat tiket ditugaskan, isi tindak lanjut.

4.6 Manajemen User (IT)

  • CRUD user: username, name, email, role, password.
  • Filter q (nama/email/username) & role; tombol aksi mobile menumpuk vertikal.

5. UI/UX

  • Filter & tombol rata tengah; tinggi konsisten; urutan mobile: cari → tanggal awal → tanggal akhir → status → aksi.
  • Placeholder tanggal: “Tgl Awal/Tgl Akhir” dengan trik toggle type focus/blur.
  • Badge status menyesuaikan di mobile; notifikasi unread kuning dan wrap teks.

6. Validasi

  • User create: username unique 3–50, name ≥3, email unique, password ≥8, role ∈ {IT,CABANG,VENDOR}.
  • User update: sama; password opsional.
  • Tiket: nomor_tiket unique; kategori string(255) nullable; deskripsi wajib; lampiran string path opsional.
  • Komentar: body wajib; attachment opsional.

8. Peta Route

Ringkas: Home→login, Profil(edit/update), Notifikasi(list/mark), Dashboard redirect peran, Ticket shared (show/comment/download/delete), CABANG (create/list), IT (list/export/my/stats + aksi), VENDOR (list/followup), IT User Management CRUD, API subcategories.

9. Ekspor

CSV dengan filter; PNG snapshot via html2canvas.

10. Keamanan

  • Semua route dilindungi auth; grup role:* dan guard inline.
  • Hapus komentar dibatasi pemilik; pembatasan di view + controller.

11. Non-Fungsional

  • Responsif mobile: filter bertumpuk, tombol lebar penuh di layar kecil.
  • Standarisasi tinggi komponen dan spacing.
  • Wrap teks notifikasi.

12. Catatan & Gap

  • Role ADMIN ada di form tapi belum ada di enum DB; perlu migrasi tambahan jika ingin diaktifkan.
  • Kolom eskalasi diubah/tambah di beberapa migrasi; kondisi akhir sudah tersedia.

13. Kriteria Penerimaan

  • Profil menampilkan dua kartu (Informasi Akun & Ubah Password) dengan input putih dan berhasil simpan.
  • Manajemen User: filter berjalan; tombol aksi nyaman disentuh di mobile.
  • Ticket CLOSED memblok komentar/hapus; server menolak bypass.
  • IT dapat menjalankan seluruh aksi lifecycle dan jejak waktu tercatat.
